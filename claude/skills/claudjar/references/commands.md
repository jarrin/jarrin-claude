### claudjar worktree merge

```
USAGE
  claudjar worktree merge [--remove] [--teardown] [--claude] <name>
  claudjar worktree merge --help

Run from the worktree you want to merge INTO (e.g. main), naming the branch to pull in.

The worktree and branch are KEPT by default — merging work up is not the same as being finished with the worktree. Add --remove to clean up in the same step: that stops the worktree's project stack, removes the directory, and safely deletes the branch (--no-teardown leaves the stack running).

On a conflict nothing is removed: the worktree and branch stay put and an interactive claude session opens in the target, seeded with both sides and the conflicted paths. --no-claude prints the paths instead.

FLAGS
     [--remove]                  After a clean merge, also remove the worktree and delete the branch     [default = false]
     [--teardown/--no-teardown]  With --remove, stop the project stack first (use --no-teardown to skip) [default = true]
     [--claude/--no-claude]      On conflict, launch claude to resolve (use --no-claude to skip)         [default = true]
  -h  --help                     Print help information and exit

ARGUMENTS
  name  Worktree / branch name to merge in (e.g. feature/x)
```

### claudjar worktree remove

```
USAGE
  claudjar worktree remove [--teardown] <name>
  claudjar worktree remove --help

The counterpart to `worktree create`, and the cleanup half of `worktree merge --remove`.

Stops the worktree's project stack (with its own assigned PROJECT_PORT), removes the directory, then deletes the branch with `git branch -d`. That safe delete succeeds only when the branch is fully merged, so unmerged work is never discarded silently — the branch survives and is reported.

Run it from another worktree; removing the one you are standing in is refused. --no-teardown removes the worktree while leaving the stack running.

FLAGS
     [--teardown/--no-teardown]  Stop the worktree's project stack first (use --no-teardown to skip) [default = true]
  -h  --help                     Print help information and exit

ARGUMENTS
  name  Worktree / branch name to remove (e.g. feature/x)
```

### claudjar help

```
USAGE
  claudjar help [--full] [--include-internal]
  claudjar help --help

`claudjar help --full` renders every command's help into a single Markdown document — the same text `--help` shows per command, so it cannot drift from the real flags.

By default the internal `api` hook commands are omitted, since they are invoked by Claude Code rather than by a person. Add --include-internal to document those too.

FLAGS
     [--full]              Print every command's help text as one Markdown document   [default = false]
     [--include-internal]  With --full, also include the internal `api` hook commands [default = false]
  -h  --help               Print help information and exit
```
